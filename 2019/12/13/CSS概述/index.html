一、CSS3概述

1、CSS发展历史

  从1990年代初HTML被发明开始样式表就以各种形式出现了，不同的浏览器结合了它们各自的样式语言，读者可以使用这些样式语言来调节网页的显示方式。一开始样式表是给读者用的，最初的HTML版本只含有很少的显示属性，读者来决定网页应该怎样被显示。

  但随着HTML的成长，为了满足设计师的要求，HTML获得了很多显示功能。随着这些功能的增加外来定义样式的语言越来越没有意义了。

​   1994年哈坤·利提出了CSS的最初建议。伯特·波斯（Bert Bos）当时正在设计一个叫做Argo的浏览器，他们决定一起合作设计CSS。

​   当时已经有过一些样式表语言的建议了，但CSS是第一个含有“层叠”的主意的。在CSS中，一个文件的样式可以从其他的样式表中继承下来。读者在有些地方可以使用他自己更喜欢的样式，在其他地方则继承，或“层叠”作者的样式, 这种层叠的方式使作者和读者都可以灵活地加入自己的设计，混合各人的爱好。

​    哈坤于1994年在芝加哥的一次会议上第一次展示了CSS的建议，1995年他与波斯一起再次展示这个建议。当时W3C刚刚建立，W3C对CSS的发展很感兴趣，它为此组织了一次讨论会。哈坤、波斯和其他一些人（比如微软的托马斯·雷尔登）是这个项目的主要技术负责人。1996年底，CSS已经完成。1996年12月CSS要求的第一版本被出版。

​   1997年初，W3C内组织了专门管CSS的工作组，其负责人是克里斯·里雷。这个工作组开始讨论第一版中没有涉及到的问题，其结果是1998年5月出版的第二版要求。到2007年为止，第三版还未完备。

​    2001年5月，W3C完成了CSS3的工作草案，主要包括盒子模型、列表模块、超链接试、语言模块、文字特效、多栏布局等模块。

​    CSS是一种网页控制技术，可以有效地对页面布局、字体、颜色、背景和其他效果实现更加精准的控制。

​   与CSS以前的版本比较，CSS3的变化是革命性的，而不是仅限于局部功能的修订和完善。尽管CSS3的一些特性还不能被很多浏览器支持，或者说支持得还不够好，但是它依然让我们看到了网页样式的发展方向和使命。

3、CSS3中的选择器

  在前面已经知道的CSS可以修改HTML中标记的样式，那它是通过什么方式去修改呢？简单的说，就是告诉CSS三个问题：改变谁？改什么？怎么改？这个时候就需要使用到选择器。选择器就是用来选择标签的方式，比如ID选择器就是通过id来选择标签，类选择器是通过类名来选择标签；改什么就是告诉CSS改变这个标签的哪一些属性，怎么改则是则是指定这个属性的值。

  举个栗子：如果要将HTML中的所有<p>标签当中的字体颜色设置为红色，就需要通过标签选择器来告诉CSS要改变的是<p>标签，其颜色为红色，清楚了这三个问题，CSS就可以很好地为我们服务啦！

3、属性选择器

  该类型的选择器针对的是带有特定属性的HTML元素。比如p[calss=cont]表示的是选择class属性值为conte的p元素。其语法包括以下几种：

语法 含义
E[attribute] 用于选取带有属性的元素
E[attribute=value] 用于选取带有指定属性和值的元素
E[attribute~=value] 用于选取属性值中包含指定值的元素，该指定的值必须是整个单词，前后有空格
E[attribute|=value] 用于选取带有以指定值开头为属性值的元素，必须是整个单词，前后有连字符

4、派生选择器/上下文选择器

  该类型的选择器会根据其位置的上下文关系来定义样式。在CSS1.0中称为上下文选择器，而在2.0中则被改为派生选择器。当说到这两个名字的时候要知道不是的就是一个选择器。而派生选择器又可分为：后代选择器、子元素选择器和相邻兄弟选择器。

  后代选择器

  可以选择某个元素的后代元素，他们之间的间隔是无限的，语法：父元素 子元素{property1:vlaue1…… }

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title></title>
        <style type="text/css">

            h1 em{
                color: #FF0000;
            }/*选择h1元素中的子元素em*/

        </style>
    </head>
    <body>
        <h1><em>HTML</em>基础</h1>
        <p><em>HTML</em>，超文本标记语言（HyperText Markup Language的简称）</p>
    </body>
</html>

效果如下：

  子元素选择器

  子元素选择器只能选择某个元素的子元素，它与之前的后代选择器最大的不同就是元素间隔不同，后代选择器选择某个父元素中的子元素，那么只要位于该父元素之内的所有子元素都是符合条件的，而子元素选择器只有对父元素而言的第一级子元素才符合条件，语法：父元素 > 子元素{property1:vlaue1…… }

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title></title>
        <style type="text/css">
            p>em{
                color: #FF0000;
            }
        </style>
    </head>
    <body>
        <p><abbr title="HyperText Markup Language"><em>HTML</em></abbr>，
        超文本标记语言，简称<em>HTML</em></p>
    </body>
</html>

效果如下：

从上面的代码和结果可以看出，因为子元素选择器针对的是父元素的一级子元素，而代码中的第一个<em>位于<abbr>当中，属于二级子元素，因此就不符合条件（如果把子元素选择器改为后代选择器，则两个<em>都符合条件），这就是后代选择器与子元素选择器的区别：元素间隔不同。

  相邻兄弟选择器

  可以选择紧接在另一个元素后面的元素。并且二者拥有相同的父元素。它与之前两个选择器不同的是它针对的是同级间的元素，并且两个元素是相邻的，还要拥有相同的元素。语法：父元素 + 子元素{property1:vlaue1…… }

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title></title>
        <style type="text/css">
            h1+p {
                color: #FF0000;
            }
        </style>
    </head>
    <body>
        <h1>HTML基础</h1>
        <p>HyperText Markup Language简称HTML</p>
        <p>该文本没有任何变化！</p>
    </body>
</html>

效果如下：

  可以看到与<h1>元素相邻的<p>元素符合条件，因此为红色，并且父元素都是<body>，而下面的<p>元素与<h1>不相邻，因此不符合条件。

5、类选择器和ID选择器

  在CSS当中应用得最多的也是最广泛的就是类选择器和ID选择器，这两种选择器在用法上大体类似，不过也有一些差别：

    类选择器：在使用该选择器时，前面会有“.”的符号，语法格式：

  .p{color:red}

    ID选择器：在使用该选择器时，前面会有“#”的符号，语法格式：

  #p{color:red}

 注意：在一个页面中可以使用多个类选择器，但是只能使用一个ID选择器！

  5、伪类和伪元素选择器

  在浏览网页时，会经常遇到一种情况，就是把鼠标放在某个元素上时，这个元素会出现一些变化，例如：当鼠标滑过导航栏时，导航栏会显示一些内容，这些特效都离不开伪类选择器，而伪元素选择器则是用来实现普通标签无法轻易修改的部分，比如一段文字中的第一个字。

  1、伪类选择器

  伪类选择器是CSS中已经定义好的选择器，因此不能随意更改名字，它是用来对某种特殊状态的目标元素应用的样式。有以下四种：

◆:link ：表示未访问时的超链接样式；

◆:visited： 表示已访问过的超链接样式；

◆:hover： 鼠标停留元素时的样式;

◆:active：正在单击元素时定的样式;

看如下代码：

                font-size: 30px;

  2、伪元素选择器

  伪元素选择器时用来改变文档中特定部分的效果样式，而这一部分是普通标签无法完成的部分。在CSS3中，有以下四种伪元素选择器：

◆:first-letter ： 对指定对象内的第一个字符起作用；

◆:first-line ： 对指定对象内的第一行内容起作用；

◆:before ： 用于在指定对象内的前端插入内容；

◆:after ： 用于在指定对象内的尾端添加内容：

4、常用属性

  1、文本的相关属性

  （1）、设置字体属性：font-family 格式如下：
  font-family:name1,[name2],[name3]

  name：字体的名称，而后面的name2和name3就相当于备用字体，当计算机中没有name1时则显示name2，当计算机中没有name2是则显示name3。如下代码：

.mr-font1{
    font-family:"宋体","黑体","楷体"
}
  （2）、设置字号属性：font-size：(length)

  length指的是字体的尺寸，也可以说是大小。由数字和长度单位组成，这里的单位可以是相对单位，也可以是绝对单位。以下表列出了几个绝对单位及其表示的含义：

长度单位 说明
in inch，英寸
cm 厘米
mm 毫米
pt point，印刷的点数，在一般的显示器中，1pt相当于1/72inch
pc pica，1pc=12pt

  常见的相对单位有px、em、ex等，下面依次介绍它们的用法：

  ◆相对单位px：

  px是一个长度单位，表示在浏览器上1个像素的大小，因为不同访问者的显示器的分辨率不同，而且每个像素的实际大小也不同，因此px也被称为相对单位。

  ◆相对单位em和ex：

  1em表示的长度是其父标签中字母m的标准宽度，1ex则表示字母x的标准高度。当父标签的文字大小发生改变时，使用这两个单位的子标签的大小也会随之改变。

  （3）、设置文字颜色属性：color：color

  color表示颜色值，颜色值的表示方法有三种：英文单词、十六进制、RGB或者HSL。如下代码示例：

h3{color:bliue};        /*使用颜色单词来表示颜色*/
h3{color:#0000ff};      /*使用十六进制表示颜色*/
h3{color:#00f};         /*十六进制的缩写*/
h3{color:rgb(0,0,255)};/*使用rgb()表示颜色*/
  (4)、设置文本的对齐方式： text-align（该属性通常有四个值：left、right、center、justify）
  (5)、设置段落缩进属性：text-indent（可通过直接定义或者百分比来表示，可以为负数）

  2、背景的相关属性

  (1)、添加背景颜色的属性： background-color|transparent

  transparent：表示背景颜色透明（默认值）。

  (2)、添加背景图片： background-image: url()

  url表示图片的地址，可以是相对地址，也可以是绝对地址

  (3)、设置图像的平铺方式： background-repeat：inherit|no-repeat|repeat|repeat-x|repeat-y

  属性值解释：

属性值 含义
inherit 从父标签继承的属性
no-repeat 背景图像只显示一次，不重复
repeat 在水平方向和垂直方向上重复显示图像
repeat-x 只在x轴方向重复显示图像
repeat-y 只在y轴方向重复显示图像 
<body style="background:url(source/HTML5.gif);">

<body style="background:url(source/HTML5.gif);background-repeat:no-repeat">

<body style="background:url(source/HTML5.gif);background-repeat: repeat-x;">

<body style="background:url(source/HTML5.gif);background-repeat: repeat-y;">

（4）、设置图像是否随着页面滚动：background-attachment：scroll|fixed

  scroll：当页面滚动时，图像跟着滚动。

  fixed：将背景图像固定在页面的可见区域。

  （5）、设置背景图像在页面中的位置：
background-position：length|percentage|top|center|bottom|left|right

  属性值解释：

属性值 含义
length 背景图像与页面水平和垂直方向的距离。
percentage 根据页面标签的宽度和高度的百分比放置背景图像
top 设置背景图像顶部居中显示
center 设置背景图像居中显示
bottom 设置背景图像底部居中显示
left 设置背景图像左部居中显示
right 设置背景图像右部居中显示

  我们可以利用length参数根据自己的需要将图片放在页面中的任何位置：

background-position:300px 100px;

【注意】当需要为背景设置多个属性时，可以使用简写“background”，然后将其个属性值写在一起，并以空格间隔。如以下代码：

.cont{
    background-image:url(../img/bg.png);
    background-position:left top;
    background-repeat:no-repeat;
}

  以上代码还可以写为以下格式：

.cont{

    background:url("../img/bg.png") left top no-repeat
}

  3、列表的相关属性

  以下为列表的几个常用属性：

◆list-style：列表简写属性。

◆list-style-image：将图像设置为列表项标志。

  5、CSS盒模型

  框模型（Box Model 也以为“盒子模型”），是CSS3中一个非常重要的概念，也是比较抽象的一个概念。文档树中的每一个元素都是以一个矩形框的形式存在的，这些框影响了元素与元素之间的距离，元素内容的位置等；而浏览器根据视觉格式化模型来将这些框布局成访问者看到的样子。这个框模型规定了处理元素内容、内边距、边框和外边距的方式。如下图是一个框模型的示意图：

从上图我们可以看出框模型类似于一个盒子，最里面的部分为元素内容，它有宽度和高度。而在往外有一条和黑色的边框，在边框与元素内容之间的称为padding（内边距），而在往外，就是最外面的那条虚线，虚线与边框之间的部分称为margin（外边距）。外边距默认是透明的，因此它不会遮挡其后面的任何元素。

  1.1、外边距（margin）

  外边距就是元素与元素之间的距离，它主要由四部分组成：margin-top、margin-right、margin-left和margin-bottom。下面以margin-top为例：

margin top:<length>|auto|;
◆auto:默认的边距（不管是那一种边距，其默认边距都为auto）

  外边距尽然有四个值，那么是不是就一定要一个一个去设置它呢？答案肯定不是啦！之前就已经讲过简写属性了，所以这里也可以使用简写属性，它分为以下几种情况：

  1、只设置一个外边距的值：当只给margin设置一个值的时候，它表示上、下、左、右的四个外边距的值都为那个数。
  2、设置两个外边距的值：当给margin设置两个外边距的值的时候，前面那个值表示上、下外边距的值，而后面那个数则表示左、右外边距的值；
  3、设置三个外边距的值：当给margin设置三个外边距的值时，前面的那个数表示上外边距的值，中间那个数表示左、右外边距的值，最后那个数表示下外边距的值；
  4、设置四个外边距的值时，它表示从顶端开始，按照顺时针的顺序，依次描述各边的值，即上、右、下、左。

  2.2、内边距（padding）

  内边距就是元素内容与对象边框之间的距离，它可通过padding属性进行设置，同样它也有四个属性值，设置的方法与前面一样（跟前面的一样，这里就不再叙述）。

  7、布局属性

  布局属性指的是文档中元素的排列显示规则，在HTML中提供了三种布局方式：普通文档流、浮动（float）和定位（position）。

  普通文档流：文档中的元素按照默认的排列规则进行排序。从上到下、从左到右。块级元素独占一行，行内元素按照顺序水平排序。元素之间不能重叠显示。

  浮动（float）：设定某个元素向某个方向浮动，元素不能重叠显示。

  定位（position）：直接定义元素在文档中位置，表现为漂浮于文档上方，脱离了普通文档流，元素可以重叠在一起，按照显示级别以覆盖的方式显示。

  浮动（float）：

  浮动可以使元素脱离普通文档流，也就是说可以浮于元素上方，因此它可能会遮挡掉后面的元素，比如以下 这个页面：

                width: 400px;
                height: 150px;
                background-color:red;
                float: left;
            }
            .d2{
                width: 400px;
                height: 150px;
                background-color: blue;

            }


        </style>
    </head>
    <body>
        <div class="d1">

        </div>
        <div class="d2">

        </div>

    </body>

  正如我们眼前见到的一样，蓝色的div跑到红色下面去了，这是因为我们给第一个（红色div）设置了左浮动属性以后，它就脱离了文档流，而下方的元素会补上来。

  浮动工作原理：当一个元素设置了浮动属性以后，它就会脱普通的文档流，浮于页面上方，该浮动的元素可以左右移动，直到该元素的外边缘触碰到带有框的元素或者另一个浮动框的边缘。（任何一个浮动元素都会被当作块级元素对待）

  影响：

  1、可能会遮挡一些正常文档流中的元素。

  2、会造成父容器高度坍塌

  可以看到。我们的两个子元素已经设置了left浮动，因此就脱离了文档流，相当于父容器中没有了元素，它的高度因此而坍塌。

  定位（position）：

  用于设置元素在页面中的位置，常见属性如下表：

属性 含义 属性值 继承
position 元素定位类型 absolute/relative/static 否
top 定位元素上外边距的偏移量 auto/长度/百分比 否
right 定位元素右外边距的偏移量 auto/长度/百分比 否
left 定位元素上左边距的偏移量 auto/长度/百分比 否
bottom 定位元素下外边距的偏移量 auto/长度/百分比 否
z-index 设置元素的堆叠顺序 auto/长度/百分比 否

  

  position：定位类型

  static：默认值，元素出现在正常位置，此时设置top等偏移量无效。

  absolute：绝对定位，生成一个绝对定位的元素，通过设置top等偏移量调整元素位置。

  relative：相对定位，生成一个绝对定位的元素，相对于正常位置而定位，不会脱离文档流。

  定位位置：定位主要依靠top、left、right、bottom四个属性来控制。

  top：定位元素顶部偏移的距离，正数向下偏移，负数向上偏移。

  right：定位元素右边偏移的距离，正数向左，负数向右。

  left：定位元素左边偏移距离，正数向右、负数向左。

  bottom：定位元素底部偏移的距离，正数向上、负数向下。

  【注】水平方向以left为准，垂直方向以top为准。
