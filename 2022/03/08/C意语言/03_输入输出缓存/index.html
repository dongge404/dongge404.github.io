输入输出缓存

windows系统运行代码

int main()
{
    for (int i = 0; i < 10; i++)
    {
        printf("Hello World!", i);
        Sleep(500);
    }
    return 0;
}

我们可以看到，每次执行一次输出语句，都会暂停500毫秒（这个时间虽然非常短暂，但是能够直观的感觉出来）。知道最后语句的输出。

linux系统运行代码

#include <stdio.h>
#include<unistd.h>
int main()
{
  for(int i = 0; i < 10; i++){
    printf("hello world!%d",i);
    usleep(1000 * 500);//使用usleep函数休眠500毫秒
}
  return 0;
}

通过运行，我们会发现，该程序会在等待5000毫秒之后一次性将所有的语句输出。

输入和输出缓存

输出缓存区

在使用printf函数时，先将指定的字符放进一个输出的缓存区，带一个特定的时间，然后一次性输出。

缓存刷新

将缓存区中的数据发送到目的地并清空缓存，这个动作称之为缓存刷新。

在windows系统下，使用了printf函数之后就会执行缓存刷新；

在linux下，使用了ptintf函数之后，会继续等待其他函数将数据写入缓存区中，直到程序执行完毕之后才执行缓存刷新；

所以，上面同一个程序不同的现象是因为缓存刷新时间点的不同所致。

行缓存

输入输出都是属于行缓存，即结束一行之后立刻执行缓存刷新。还有另外一种是完全刷新，这种刷新只有当缓存区中数据被写满才会执行缓存刷新。对于行缓存，我们只要在其输出语句中加上一个”\n”即可立刻刷新缓存。将上述的linux代码中加入一个”\n”，并执行代码：

我们会发现它的执行和Windows系统上一样了，这是因为当linux系统中，执行第一个printf函数时，字符串参数中包含一个换行符”\n”,此时立刻缓存刷新，将输出缓存中的数据输出到控制台上，并继续往下执行（休眠500毫秒）。

输出缓存是一种系统特性，不是一种函数特性！

输出缓存区

比如scanf、getchar等输入函数存在输入缓存，因为这些函数属于阻塞函数，也就是当直接执行到该函数时，会处于阻塞状态，等待用户输入内容，并回车（”\n”）。这些函数就会把用户输入的内容送到输入缓存区。接下来，输出函数就会去输出缓存中读取数据，之后删除已读取的数据，并解除阻塞状态。=，继续往下执行。

再来看一下下面的代码：

char c1, c2;
c1 = getchar();  //输入一个字符，保存在c1变量中
putchar(c1); //输出该字符
c2 = getchar();  //输入两一个字符，保存在c2变量中
putchar(c2); //输出另一个字符

结果：

结果并不是我们想象的那样，这是因为在第一个getchar函数中，我们输入一个字符D，然后回车，此时该函数就把“D\n(换行)“两个字符放进输入缓存区，此时该函数不在阻塞状态，继续往下输出D字符 ，此时输入缓存区中含有\n，那么getchar函数不在进入阻塞状态，直接读取\n，之后继续输出，此时c2中读取的是\n，所以就会输出一个换行。

将上面的代码做以下改进：


    char c1, c2;
    c1 = getchar();
    putchar(c1);
    getchar();  //同于吸收输入缓存区中的\n
    c2 = getchar();
    putchar(c2);

将输入缓存区中的\n获取之后，就没有内容了，在使用输入函数时，它就又进入了阻塞状态。继续等待用户输入，然后输出 即可。

不带缓存的函数

getche函数

while (1)
    {
        char c;
        c = _getch();  //键盘输入一个字符并赋值给c变量
        putchar(c);  //通过putchar函数将字符输出到控制台
        if (c == 'q')
        {
            break;
        }
    }

getch函数是一个无缓存的getchar函数，该函数和getchar最大的不同就是没有缓存区，即当输入一个字符 之后，直接通过putchar打印到控制台。

getche函数

while (1)
    {
        char c;
        c = _getche();  //键盘输入一个字符并赋值给c变量，然后打印
        if (c == 'q')
        {
            break;
        }
    }

该函数相比较上面的函数，多了一个输出的功能，当输入吧一个字符，就立即将字符在控制台打印。
