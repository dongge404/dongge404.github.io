Vue-Cli

这是官方名称，也就是我们俗称的脚手架。说到脚手架，我们可能会想到下面这张图中的东西：

看到这个东西，其实也就想到在建筑高楼的时候，常常会在外面设计这样的一个东西，这样的东西就称之为脚手架。它是整个工程的基础，如果没有这个定西，我们就很难去建造一栋高大的楼层。同样，在Vue开发中它也是如此，一个项目的实施和运行固然离不开脚手架，只不过这里所的脚手架为Vue-CLI。

Vue-Cli安装

进入官网，我们找到安装，然后在自己的电脑的命令窗口中输入下面命令：

npm install -g @vue/cli

我们在安装之前，应该先输入这样的命令：

npm config set registry https://registry.npm.taobao.org

该命令会让我们的安装速度快一些。

安装完毕之后，查看版本信息：

vue -V

项目文件结构

在正式使用脚手架之前，我们需要对项目的结构有一定的了解，打开项目，文件内容：

对应的结构视图：

├── node_modules   依赖包的目录
├── public
│   ├── favicon.ico: 页签图标
│   └── index.html: 主页面
├── src
│   ├── assets: 存放静态资源
│   │   └── logo.png
│   │── component: 存放组件
│   │   └── HelloWorld.vue
│   │── App.vue: 汇总所有组件（入口组件）
│   │── main.js: 入口文件
├── .gitignore: git版本管制忽略的配置
├── babel.config.js: babel的配置文件
├── package.json: 应用包配置文件 
├── README.md: 应用描述文件
├── package-lock.json：包版本控制文件

ref属性

该属性用于标记页面上的元素或者组件，设置了该属性的元素或者组件。会在自身组件的实例对象上的$refs身上绑定。属性值就是被绑定元素或者组件的属性名，属性值就是该元素或者该组件，

<h2 ref="title">你好,{{ name }}</h2>

<School  ref="school"/>

<Hello ref="hello"/>

h2为App组件中的标签，School和Hello为组件标签，当打印App组件的组件实例对象时，可以在其$refs属性中找到这些标记了这些属性的 元素或者组件：

props配置对象

该配置对象用于组件实例对象中，主要作用是接收外面传递过来的一些数据。在App组件中传递一些数据：

<template>
  <div>
    <h2 ref="title">你好,{{ name }}</h2>
    <School address = "昆明" son = "热带作物学院"/>
    <Hello/>
  </div>
</template>

在App组件中给School组件标签传递address和son数据

export default {
  name: "School",
  data() {
    return {
      SchoolName: "云南农业大学",
    }
  },
  components: { Student },
  props: ['address','son']
}

在School组件中使用props配置对象接收，它的值可以是一个数组、也可以是一个对象。数组中的元素均为字符串，每个字符串代表向父组件中接收的数据（父组件中传递的属性值）

当然，也可以写成一个对象：

  props: {
    address: {
      type: String,  // 定义数据类型为字符串
      required: true   // 规定该数据为必传递
    },
    son: {
      type: String,
      default: "某带某物学院"  // 当没有传递该数据的默认值
    }
  }

mixin混合模式

当两个组件中同时拥有同一个方法时，我们可以把这个函数单独拿出来，封装到一个名为mixin.js的文件中，该文件和main.js同级。

mixin.js文件：

export const  a =  {
      methods: {
        showName() {
          alert(this.SchoolName)
        }
      }
}
