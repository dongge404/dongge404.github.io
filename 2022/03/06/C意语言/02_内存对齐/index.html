C语言中的数据存储

我们都知道，计算机只会处理二进制的数据，用户通过一些输入设备（鼠标、键盘等）将数据输入到计算机当中，计算机会把这些数据存放到主存当中。而主存是计算机硬件中不可或缺的一个部分。存储在主存中的数据会对应着一个编号（该数据在主存中存放的位置），利用这个位置，就可以找到并读取这个数据。我们把这个内存的地址称为数据在内存中的存放位置。

通过上面的图片，我们可以理解到，数据在计算机内存中的存储大致的情况，我们要想使用或者读取这些数据就必须知道这个数据存放的位置。不同的数据类型，它在内存中所占用的空间也是不一样的，下面将说明几种常用数据类型在内存中的大小：

整型数据

char类型的数据

char关键字用于声明一个字符类型的整型数据，这里为什么会把字符类型归结为整型数据呢？

因为字符在计算机内存中也是以整数形式存储的，而计算机会先将字符转换为对应的整数值，然后在将这个整数转换为对应的二进制数放到计算机中存储。而这个字符和整数之间的对应关系，采用的就是ASCII码来实现的。每个字符会唯一对应一个整数，因此char类型的数据本质上也属于整型。

char类型的数据在内存中占用一个字节的大小。

short类型的数据

short类型的数据称为短整型数据，在内存中占用2个字节的大小。

int类型的数据

int类型的数据为整型数据，在内存中占用4个字节的大小。

long类型的数据

long类型的数据称为长整型数据，在内存中占用4个字节的大小

long long类型的数据

long long类型的数据称为更长的整型，在内存中占用8个字节的大小。

浮点型数据

float类型的数据

float类型为单精度浮点型数据，在内存中占用4个字节的大小。

double类型的数据

double类型的数据为双精度浮点型数据，在乃村中占用8个字节的大小

精度越高，所表示数值的范围也就越广

结构体大小的计算规则

既然不同的数据类型占用的空间不同，那么结构体的大小究竟应该怎么计算呢？因为结构体中的成员可以使一些不同类型的成员，对于这些占用不同大小空间的成员，结构体又应该怎么计算它的大小呢？我们先定义一个结构体：

struct {
    char name[10];
    int age;
    double height;
    double weight;
}person;

上面定义了一个结构体变量person，其中有四个成员，分别是char、int和两个double。我们可以猜测这个结构体的大小会不会是这些成员占用总空间的大小呢？如果实在这样的话，上述结构体占用的大小就应该是30个字节。我们通过使用sizeof关键字来测量一下这个结构体的大小：

printf("%d\n",sizeof(person));  //32

结果为32。这就证明我们的猜想是错误的，因为我们预期的结果是30，所以可以得出结论：结构体的大小并不等于所有成员大小之和。那么它到底是根据什么来计算结构体的大小的呢？

内存对齐

没错，结构体的大小正式运用了内存对齐的规则来进行计算的。我们先来熟悉一下它的规则：

  1. 结构体中的第一个成员放在结构体的首地址（偏移位置为0的地方）处。
  2. 第二个成员之后的所有成员，均会被放在相对于首地址偏移的一个对齐数（成员大小和默认对齐数中的较小值）的整数倍的地方。
  3. 整个结构体的大小就是所有结构体成员对齐数中最大的对齐数的整数倍
  4. 如果嵌套了结构体，嵌套的结构体对齐到它最大对齐数的整数倍，结构体的大小（包含嵌套）就是最大对齐数的整数倍。

​

在VS中，默认对齐数为4，在linux系统中，默认对齐数为8

在上面的图面中，我们清楚的看到哥哥结构体成员在内存中的存放情况，第一个成员为字符数组，有10个元素，因此占用10个字节，其中第一个元素的地址即为该结构体的首地址，从这个地址开始偏移，往后偏移值为12时，正好是第二个成员对齐数的整数倍，因此会往后两个字节，继续存放下一个int成员，到了第三个成员也是一样的，因为是double类型的成员，对齐数为8，因此需要找到8的倍数的偏移，正好为后面的16，因此继续存储，后一个double同理，最终得到这个结构体总共占用了32个字节，其中有两个字节被闲置。

可以看出结构体中成员并不是连续村存放的，为什么不连续存放了？

在大多数的编译器中，规定访存为4个字节（一次访问存储器能读取4个字节的内容），这样针对上图，我们对int类型的结构体只需要访问一次就能够全部得出，如果采用连续存储，则需要访问两次存储器，这样就大大降低了程序的执行效率。所以说内存对齐是一种以“牺牲空间换取世间”的方法。
