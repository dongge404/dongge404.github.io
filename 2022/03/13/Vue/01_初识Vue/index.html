初识Vue

Vue是一个用于构建用户界面的渐进式js框架。采用这个框架构建出来的网站具有更强的交互性和数据同步的能力。是目前国内使用最广的前端框架。官网为：点我进官网，目前该版本已经更新到了3.x。当然还是需要先学习2.x的版本，之后接触新的版本才能更好的了解版本的之间的区别和联系。

学习vue之前的准备

正如官网所说的，学习vue的前提就是需要我们掌握前端中的一些中级知识，只有了解了这些，我们才能更好的推进学习。因此如果没有基础的我们需要先学习一遍前端的基础知识模块（html、css、javascript）。

安装Vue

学习Vue的第一步就是先了解它是一个什么东西，之前已经说过它是一个框架，因此是一个已经封装好的js文件，我们需要下载这些文件才能学习Vue。

在官网中，我们可以看到Vue会存在两个版本，一个为开发版本，另一个为生产版本。这里需要解释一下，开发版本就是程序员在进行开发过程中使用的版本，因此，在学习过程中我们也是使用这个版本。而生产版本，是项目上线之后才使用的，也就是说它的代码是经过压缩的。

当我们下载了两个版本之后，如上图，第一个文件是生产版的，第二个是开发板的。我们把这两个文件放进一个文件夹中，并把这个文件夹放到我们项目的根目录下。我们下载文件的原因是我们通过使用script标签来引入这个文件，就可以在这个文件中使用vue了。

  <!-- 引入Vue -->
  <script src="../js/vue.js"></script>

第一个Vue项目

在我们成功引入Vue之后，我们的第一个案例肯定就是hello world了，接下来我们写下面的代码：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <!-- 引入Vue -->
  <script src="../js/vue.js"></script>

  <title>第一个Vue案例</title>
</head>
<body>
  <!--Vue实例对象管理的容器-->
  <div id="root">
    <h1>Hello {{ name }}</h1>
    <h1>Hello {{ py.toUpperCase() }}</h1>
    <!-- {{}}是vue的插值语法 -->
  </div>
    <script>
      Vue.config.productionTip = false; //阻止vue开发版本的提示内容
      //创建Vue实例
      new Vue({
        el: "#root", //用于指定当前的Vue实例为哪个容器服务 值为字符串的css选择器
        data: {
          name: "云南农业大学热带作物学计算机",
          py: "ynnydx"
        } //data的值是一个对象（暂时），里面保存的是容器对应的数据
      })


      /* 
      注意：
      1、Vue是一个构造函数，所以一定要使用创建实例对象的new关键字
      2、Vue实例对象一次只能管理一个容器对象，不能管理多个容器
      3、{{}}是Vue中的语法，里面可以写js表达式
      4、当修改data中各个对象的值时，页面的数据也会发生变化
      */
    </script> 
</body>
</html>

运行结果如上图，确实在页面上输出了内容。我们可以看出，页面中的hello是一个标签中的内容，会自然显示，但是后面的数据却并不在页面上直接写出来，而是把这些数据放在了一个Vue实例对象中的data对象中。

可能到这，我们只是看到代码变得更加复杂了，并没有感觉到这些数据的神奇之处，我们继续往下看。

      new Vue({
        el: "#root", //用于指定当前的Vue实例为哪个容器服务 值为字符串的css选择器
        data: {
          name: "云南农业大学",
          py: "ynnydx"
        } //data的值是一个对象（暂时），里面保存的是容器对应的数据
      })

我们修改实例对象中的name数据，当我们保存，你会神奇的发现，页面的值也会随之被修改：

我们并没有改变html中的内容，而是直接修改实例对象中的数据，这就是Vue最有特色的地方——数据驱动视图

插值语法

在前面的代码中，我们在html代码中写入了花括号这样的一个符号。这和符号正是Vue中一个非常重要的语法——插值语法。在之前的js语法中，我们是是使用命令式编码，而Vue采用的是声明式的编码。

命令式和声明式的编码称为编程范式，就是一种采用不同风格代码的程序执行模式，命令式指的是代码一句一句的执行，即对于一个程序来说，就是一步一步的往前走；而声明式指的是对于一个程序，做一件事只要给定一个声明，都是一气呵成，我们不必要去一步一步的实现它。这样的好处是能很大程度的提高编程效率。

对于Vue中的插值语法，它主要用于解析标签体内容的东西，比如上述代码：

  <div id="root">
    <h1>Hello {{ name }}</h1>
    <h1>Hello {{ py.toUpperCase() }}</h1>
    <p>{{1 + 1}}</p>
    <p>{{Date.now()}}</p>
  </div>

在html的标签中，使用了这个语法，里面的name和py实际上是Vue实例对象中定义的两个数据（变量），只有将这样个变量放在一对嵌套的花括号内部，才能被解析，否则将会报错。其实也看到，这个嵌套的花括号中不仅仅可以放置这些数据，还可以放置一些js的表达式。（这里不是js语句）。

el和data的两种写法

我们需要知道的是，在引入了Vue之后，该代码中其实 就已经定义了一个Vue的构造函数，这也就是为什么我们需要实例化对象的原因，通过Vue构造函数实例化的对象称为vue的实例对象，它是整个Vue中的核心。下面是该对象的定义：

const vm = new Vue({
    el: "#root", //el是实例对象中第一个配置属性，用于管理制定的html容器
    data: {
        name: "张三",
        age: 20
    } //data是实例对象中的第二个配置对象，用于保存一些数据，我们可以通过插值与法将这些数据在页面中进行展示
})

其实在上面的代码中，我们定义的el和data还有其他 写法：

const vm = new Vue({
   data(){
       return {
           name: "张三"
           age: 20
           ...
       }
   }
})
x.$mount("#root")

在上面的写法中，因为在实例对象的原型上存在一个函数$mount()用于挂载指定的html元素，data也不是之前的对象了，而是一个函数，函数再次返回一个对象。（这种写法在后面的组件中是必须使用的）

MVVM

mvvm是一种软件架构，vue正是借助了这种架构，将数据和视图连接在一起。

m：model。 模型层，用于放置一些基础的数据

v：view。 视图层，即直接面向用户的页面，在页面中直接展示数据

vm： viewmodel 视图模型层，是整个vue的核心，正是有了它，才能是vue拥有这么多强大的功能

对上图进行一个简单的分析：

1、view层中，当数据发生变化时（数据发生变化是因为用户的交互，可能输入了某些数据），在vm中，通过对DOM的监听，从而告知vue页面的数据发生变化，需要通过同步更新到model层。

2、当model中的数据发生变化时，由于通过vm实现了数据绑定，因此，慧将更改的数据同步更新到页面中。

3、上面的两条线就足以使得数据之间可以相互绑定了，再通过一些常用的指令，可以使得vm具有更加强大的功能 ，因此可以把中间的这个视图模型层看作时一个Vue的实例对象。

下面，我通过一个例子来展示这个架构的原理：

    const vm =  new Vue({
      el: "#root",
      data: {
        school: "云南农业大学",
        address: "普洱"
      }
    })

上面的代码表示在Vue实例中，定义了两个数据，school和address，这两个数据将被保存在model层中，也就是上面的最右边的那个东子当中，当我们运行代码时：

我们看到页面上内容属于视图层的内容。

我们点击右下角的address中的内容，将其改为“昆明”，然后点击右侧的按钮保存，这是我们观察页面：

我们会发现，随着我们在后台更改数据，随着页面也会发生变化，这就是之前所说的数据驱动视图的更新，这也是Vue最具特色的地方。当然school数据也是同理，只要我们改变数据，他就会被Vue（也就是视图模型层）监听到，然后对应将视图上的数据也改变。

这里我们说的时上面图中的从model到view之间的路线，那从view到model之间的路线其实还需要借助一个指令：v-model来实现。

  <div id="root">
    数据单向绑定: <input type="text" :value="name"><br>
   <!--  数据双向绑定: <input type="text" v-model:value="name"> -->
    数据双向绑定: <input type="text" v-model ="name">
  </div>

一个文本框实现数据的单向绑定，这个其实就是从model到view之间的转变，而我们通过代码，可以发现，第二个文本框中添加了一条新的命令—v-model，该命令用于实现数据的双向绑定，及既可以实现model——view（本身具备的特性），还可以实现view——model之间的转换。
