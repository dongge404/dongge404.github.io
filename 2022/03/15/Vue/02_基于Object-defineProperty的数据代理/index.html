基于Object.defineProperty的数据代理

Object中的defineProperty函数

在js中，我们已经学过了Object对象中许多函数，今天我们就来介绍一下它的defineProperty函数。我们可以基于这个函数来实现基本的数据代理。

数据代理可以理解为一种通过操作一个对象去修改或者读取另一个对象中的属性或者方法的行为，在Vue中，可以把这种行为称为数据劫持，它是一种基于Object.defineProperty实现的功能，当对该对象中的某些属性进行读取或者修改时，会被某些指定的函数所劫持，从而进行一些其他的操作。

defineProperty

该函数用于在指定的对象中定义属性，有三个参数，第一个参数为指定的对象，第二个参数为需要定义的属性（字符串），第三个参数是一个配置对象（{}），在该配置对象中，有两个函数，get和set，下面通过一段代码来说明该函数的作用：

    let obj = &#123;
      name: "张三",
      age: 23
    &#125;
    // 通过defineProperty函数为obj对象定义一个sex的属性
    Object.defineProperty(obj,'sex', &#123;
      // 该函数会在访问obj对象中的sex属性时调用
      get() &#123;
        console.log("sex属性被访问");
      &#125;,
      // 更改函数会在修改obj对象的sex属性时调用
      set(val) &#123;  //该形参就是被修改之后的值
        console.log("sex属性被修改，修改之后的值为" + val);
      &#125;
    &#125;)

一个简单的数据代理案例

    let obj1 = &#123;x:200&#125;
    let obj2 = &#123;y:300&#125;
    // 通过obj2来获取或者修改obj1中的x属性
    Object.defineProperty(obj2,"x",&#123;
      get()&#123;
        return obj1.x
      &#125;,
      set(value)&#123;
        obj1.x = value
      &#125;
    &#125;)

上面的代码中，有两个对象obj1和obj2，我们通过使用defineProperty函数为obj2添加一个x的属性，在读取该属性时，返回obj1的x属性，同理，在set函数中，通过修改obj2的x属性来将修改之后的属性赋值给obj1.这样就完成了一个简单的数据代理案例。

Vue中的数据代理

在知道的数据代理的大致原理之后，我们可以发现，在Vue中其实也存在数据代理。

    var vm=new Vue(&#123;
      el:'#app',
      data: &#123;
        school: "云南农业大学",
        address: "昆明"
      &#125;
    &#125;)

在vm实例对象中保存了school和address两个对象。

在控制台输出vm实例，我们可以在该实例上找到我们定义的school和address两个数据（红色框中的部分），其实这两个数据一开始并不是直接拿到的，而是从另一个对象（_data 绿色框部分）中获取过来的，vm是一个对象，从这个对象中获取另一个对象中的值就属于数据的代理了。当我们打开_data时，会发现有以下结构：

我们从中可以看出，这两个数据（school和address）在_data对象中作为两个属性，除此之外，还有为每个属性配置了两个函数：get和set。这就是关键了，其实这个数据代理的实现就来源于之前所说的defineProperty函数实现的原理。以school属性为例，当访问该对象的school属性，get函数调用；当修改school属性值时，set函数调用，里面其实做了一些操作，将这些属性或者属性的值给了Vie实例对象，此时Vue实例对象才能拿到这些数据，并最终放在自己身上。（至于函数怎么具体实现这不必探究）。

其实在Vue中，数据代理做了斜面两件事：

  1. 加工data中的数据——这里所说的加工就是指将这些数据变为响应式数据
  2. _data = data —— 此时把data中的数据赋值给_data对象（只有对data数据进行处理之后才能进行这一步）
