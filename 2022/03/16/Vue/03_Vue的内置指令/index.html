Vue中的一些内置指令

之前就已经说过，Vue是一种声明式的编程方式，即只需要给定元素一些特殊的声明，Vue就能够进行轻松的工作。

v-text

该指令用于在指定的标签中，解析文本内容。

<h2 v-text = "name">你好</h2>    

<script>
var vm=new Vue({
      el:'#app',
      data: {
        name: "云南农业大学",
        str: "<h2>你好啊!</h2>"
      }
    })
</script>

这个指令确实可以在指定的模板中解析一些文本，但是它的缺陷就是，它会把模板之前的内内容给替换掉，而且遇到带有标签的模板，它也不会自己解析模板中的标签：

  <div id="app">
     <h2>你好，{{ name }}</h2> 
     <h2 v-text = "name">你好</h2>
     <h2 v-text = "str"></h2>

  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        name: "云南农业大学",
        str: "<h2>你好啊!</h2>"
      }
    })
  </script>

大多数情况下，我们还是使用插值语法来解析内容，因为虽然写法上比插值语法要简单，但是比较v-text来说会比较“智能”。

v-html

该声明比较上一个来说，可以解析模板中的html，但是和v-text一样也会覆盖之前的内容。

  <div id="app">
     <h2>你好，{{ name }}</h2> 
     <h2 v-html = "str"></h2>

  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        name: "云南农业大学",
        str: "<em>你好啊!</em>"
      }
    })
  </script>

该指令在使用上，可能存在一定的安全性，可能会遭到XSS攻击，所以该指令一定要慎用！

v-cloak

该指令主要用于屏蔽一些未解析的模板数据。有时候由于网络的不稳定，导致Vue延迟工作，此时页面上的一些模板中需要使用Vue进行解析的数据会被显示，为了隐藏这些数据，可以使用该命令。

我们使用计时器函数模拟网络的不稳定，假设在1s之后，Vue才能工作，就会出现下面的现象 ：

  <div id="app">
     <h1>{{ name }}</h1>
  </div>

  <script>
    // 使用定时器模拟网络情况
   setTimeout(() => {
      var vm = new Vue({
       el: '#app',
       data: {
         name: "云南农业大学"
       }
     })
   }, 1000);
  </script>

v绑定了v-loak命令的标签，会在Vue开始工作时自动消失，利用这个特性，在配合属性选择器，就可以将Vue工作之前将该标签隐藏，这样就不会出现未解析的模板数据了 。

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
  <script src="../js/vue.js"></script>
  <style>
    [v-cloak] {
      display: none;
    }
  </style>
</head>
<body>
  <div id="app">
     <h1 v-cloak>{{ name }}</h1>
  </div>

  <script>
    // 使用定时器模拟网络情况
   setTimeout(() => {
      var vm = new Vue({
       el: '#app',
       data: {
         name: "云南农业大学"
       }
     })
   }, 1000);
  </script>

v-once

该指令用于一次性渲染指定模板数据，之后就不再渲染和解析

  <div id="app">
    <h2 v-once>初始的值为： {{ n }}</h2>
    <h2>当前的值为: {{ n }}</h2>
    <button @click = "n++">+1</button>
  </div>

v-pre

该指令用于标记某个模板不会被Vue所解析，注意，这个指令一定要慎用，特别是一些带有需要Vue解析数据的模板，一定不能加该指令，用该指令可以给一些不带任何解析数据的静态数据绑定，这样可以提高Vue的解析效率。

    <h2>Vue学习很简单的，小可爱</h2>
    <h2 v-once>初始的值为： {{ n }}</h2>
    <h2>当前的值为: {{ n }}</h2>
    <button @click = "n++">+1</button>

v-on

该指令用于向指定的元素添加事件。

写法一：

 <button v-on:click = "showInfo1">点击欢迎</button>

写法二：

 <button @click = "showInfo1">点击欢迎</button>

指令的值为一个方法名，该方法被定义在一个methods配置对象中（该配置对象和data、el等都是同级的）

methods: {  // 配置对象中为事件函数

    showInfo1: fucntion() {
        //函数体
    }

    // 根据对象的增强写法，可将上面的函数写为：
    showInfo1() {
        // 函数体
    }
}

上面的代码表示点击按钮，触发鼠标点击事件，然后去执行methods中的showInfo函数。

methods中定义的所有函数，其this指向永远都是Vue的实例对象

该函数也可以传递参数：

<button @click = "showInfo2($event,101)">点击欢迎</button>

在上面的代码中，传递了两个形参，$event和101，第一个参数代表接收的对象是事件对象，第二个参数是一个具体的值

showInfo2(e,number) {
       console.log(e);  // e是事件对象
       console.log(number); // number是一个具体的值，接收调用时传递的101
}

值得注意的是，如果我们不传递参数，在函数中默认也会接收一个参数，该参数用来保存事件对象，当需要事件对象和其他参数时，调用时必须使用$event来接收事件对象。

事件修饰符

enter

enter是一个键盘事件常用的修饰符，比如按下回车键，输出”我爱你”

<input type="text" placeholder="请输入信息" @keydown.enter = showInfo>

<script>
    // 部分代码
    methods: {
        showInfo() {
          console.log("我爱你！！！");
        }
      }
</script>

可以看出，事件修饰符是采用@事件名.事件修饰符的形式使用，当按下回车键就会触发上面的函数。

同理，其他键盘事件修饰符还有delete、tab、Ctrl、space、esc等。

stop

该修饰符用于阻止事件冒泡

<div class="father" @click = "clickF">
   <div class="son" @click.stop = "clickS"></div>
</div>

点击绿色子盒子，触发子盒子事件函数，就不会触发父盒子的函数

同理，还有一个类似的用关于阻止浏览器默认行为的prevent。

v-bind

用于动态绑定属性值。

<body>
  <div id="app">
    <h2 v-bind:name="name">哈哈哈</h2>
  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        name: "123"
      }
    })
  </script>

使用v-bind指令对name属性进行动态绑定，此时Vue就可以解析该属性后的字符串name，并把data中的name的值赋值给该属性：

它也有结简写形式：

<h2 :name="name">哈哈哈</h2>

v-if

改变指令用于根据条件渲染模板。即条件渲染的指令

<h2 v-if = "false"> {{ name }} </h2>

上面的模板不会被解析，因为表示的结果为false：

只有当结果为true或者符合某个条件才显示：

<h2 v-if = "true"> {{ name }} </h2>

当然我们也可以将该命令和v-if、v-else if搭配使用：

<h2 v-if = "1 + 1 === 1"> {{ name }} </h2>
<h2 v-else>条件不满足</h2>

但是，不可以将上面的代码分开：

<h2 v-if = "1 + 1 === 1"> {{ name }} </h2>
<h2>哈哈哈哈</h2>
<h2 v-else>条件不满足</h2>

我们也可以将的一组结构作为一个条件是否满足所展示的内容：

<div v-if = "n === 0">
    <h2>我</h2>
    <h2>爱</h2>
    <h2>你</h2>
</div>

上述代码可以改进的地方在于，我们不必构造一个全新的标签出来，这样会破坏模板原来的结果，所以可以使用template模板标签来代替div：

     <template v-if = "n === 3">
         <h2>我</h2>
         <h2>爱</h2>
         <h2>你</h2>
     </template>

v-show

该指令也是用于条件渲染，只不过和v-if不同的是，它是通过修改DOM元素的display属性来动态展示元素，而v-if则是直接将元素删除。

v-for

哎指令用于列表的渲染，类似遍历。

  <div id="app">
    <h2>遍历数组</h2>
    <ul>
      <li v-for = "(p,index) in persons" :key = "index">
        {{index}}-{{ p.name}}-{{ p.age }}
      </li>
    </ul>
    <hr>
    <h2>遍历对象</h2>
    <ul>
      <li v-for = "(val,key) in cars" :key = "key">
        {{val}} - {{key}}
      </li>
    </ul>
  </div>

<script>
    var vm=new Vue({
      el:'#app',
      data: {
        persons: [
          {id:'001',name:"张三",age:18},
          {id:'002',name:"李四",age:20},
          {id:'003',name:"王五",age:22},
        ],
        cars: {
          name: "法拉利",
          color: "红色",
          price: 120
        }
      }
    })
 </script>

该指令多数用于数组的遍历，以上面的代码为例，使用指令遍历了一个数组和一个对象。

该指令中共有两个参数，即遍历的元素和对应的索引（索引从0开始），如果使用两个参数则需要加上括号，使用其中一个则不需要添加

如果是遍历对象，第一个参数为属性值，第二个参数为属性。遍历的次数由数组中的元素或者对象中的属性来决定。

关于列表中的key属性

需要说明的是，在遍历列表的过程中，我们往往会忽视一个问题，那就是列表中的li元素上的key属性。该属性虽然不写不会有什么报错，但是基于Vue的底层原理，建议在数据项的元素上绑定一个key属性。值可以是index，也可以对象中的id值。

基于列表渲染的key属性值的设置
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
  <script src="../js/vue.js"></script>
</head>
<body>
  <div id="app">
    <button @click = "add">添加一个老李</button>
    <ul>
      <li v-for = "p,index) in arr" :key = "index"> 
        {{ index }} - {{ p.name }} - {{ p.age }}
        <input type="text">        
      </li>
    </ul>
  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        arr: [
          {name: "张三",age: 12,id:'001'},
          {name: "张四",age: 22,id:'002'},
          {name: "张五",age: 32,id:'003'}
        ]
      },
      methods: {
        add() {
          const p = {name:"老李",age:42,id:"004"}
          this.arr.unshift(p)
        }
      },
    })
  </script>
</body>
</html>

可以看出其实并没有问题，但是在这个代码的背后，其实存在一个很大的问题！注意我们这个例子研究的是key的值，所以先使用index，即索引值来作为key的值，所出现的问题。

我们在每个文本框中输入对应节点的数据：

我们现在点击按钮，我们想的是当在该数组中的最前面添加了一个老李之后，这些文本框中的数据也应该是意义对应的，只有新增加的列表项中的文本框没有数据，但是：

通过观察，会发现，张三的数据跑到老李的文本框中；张四的数据跑到张五文本框中；张五的数据跑到张四文本框中。这其实是一个严重的问题！如果我们使用index作为key的值，结构上确实没啥问题，但是一旦结构中有数据，不就全部乱套了吗？接下来通过一张图来解释一下这个原理：

1、初始化数组中的三条数据，生成对应的3个虚拟DOM

2、将这些虚拟DOM转换为真实DOM

3、在每个真实DOM上添加对应数据（这些数据是被写在真实DOM上，并非虚拟DOM）

4、点击按钮，在数组的前面添加一条新的数据

5、此时总共有四条数据，添加的这条数据的index为0,并生成对应的额虚拟DOM（总的有4条）

6、使用diff算法做虚拟DOM的比较，注意该算法是根据key的值来逐一比对的，即0—0、1—1等。

  • 0（老李）— 0（张三），进行对比，发现数据不同，该算法就会再生成一个真实DOM（老李），继续比较后面的文本框（比较的是虚拟DOM，因此是没有任何数据的），发现一模一样，就把之前生成的文本框（真实DOM）拿过来复用，就连带数据也拿过来了，老李中的文本框其实是复用张三的，因此带有的就是张三的数据。
  • 1（张三）—1 （张四），进行对比，发现数据不同，该算法就会再生成一个真实DOM （张三），其实之前已经有了张三的展示DOM了，因为是index值的比较，所以导致数据不同，所以就又把该DOM重新生成一遍。
  • 2（张四）— 2（张三），同理。
  • 3（张五）— 没有对比的虚拟DOM，新生成一个真实DOM ，其文本框也是新生成的，数据为空。

7、本该可以复用的DOM全部重新生成，聊率大大降低。

8、如果不使用key属性，结果默认还是将index作为key的值

9、使用id：

因为diff算法会根据id确定的唯一元素进行对比，实际上使用id只会生成一个真实DOM，其他的均为复用，而且数据还不会出错。
