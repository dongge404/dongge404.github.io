过滤器

在Vue实例的配置对象中，有一个filters的节点配置对象。

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>过滤器</title>
  <script src="./js/vue.js"></script>
</head>
<body>
  <div id="app">
      <!-- 在插值与法中使用管道符 | 之后填写一个过滤器中的函数 不加括号 -->
     <h2> {{ n | toChange}}</h2>
  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        n: 1
      },
      // 过滤器节点配置对象
      filters: {
        toChange(value) {
          return value * 10
        }
      }
    })
  </script>
</body>
</html>

过滤器本质上是一个函数。在插值语法中，使用管道符|把里面的内容和函数分割，该函数会把管道符前面的内容作为函数的参数传递，value其实接收的是n的值（1），函数返回一个结果（n * 10），过滤器函数就会把这个结果覆盖到当前的插值语法中。

上述是过滤器的最基本也是最简单的工作原理。当存在多个过滤器时：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>过滤器</title>
  <script src="./js/vue.js"></script>
</head>
<body>
  <div id="app">
    <!-- 在插值与法中使用管道符 | 之后填写一个过滤器中的函数 不加括号 -->
     <h2> {{ n | toChange1 | toChange2 | toChange3}}</h2>
     <!--  -->
  </div>

  <script>
    var vm=new Vue({
      el:'#app',
      data: {
        n: 1
      },
      filters: {
        toChange1(value) {
          return value * 10
        },
        toChange2(value) {
          return value * 10
        },
        toChange3(value) {
        return value * 10
        }
      }
    })
  </script>
</body>
</html>

第一个过滤器会把返回的结果10传递给第二个过滤器，经过计算，又把结果100传送给第三个过滤器……以此类推，只要有多个过滤器，上一个过滤器会把计算的结果传递给下一个过滤器，知道最后一个过滤器执行完，就会把结果放到当前的模板中解析。

上面定义的过滤器是一个局部过滤器，如果想在其他实例中使用该过滤器，可以定义一个全局的：

    Vue.filter('toChange1',function(value) {
      return value * 10
    })
    Vue.filter('toChange2',function(value) {
      return value * 10
    })
    Vue.filter('toChange3', function (value) {
      return value * 10
    })

自定义指令

我们在之前已经学过很多的指令，v-for、v-if、v-show、v-bind等等。这些指令都有一个共同点，那就是我们只需要给元素绑定这些指令，那么它们的功能是Vue帮助我们实现的。并不是我们自己去实现这些功能，这叫做系统指令，当然，功能也可以由我们来实现，这类的指令称为“自定义指令”

在vm实例对象中，可以在与data同级的对象directives中定义自定义的指令：

directives: {
   big(element,binding) {
      // 将值解析并放在页面上(该值被放大10倍)
      element.innerText = binding.value * 10
    }
 }

节点中也是配置一个函数，该函数中用来写一些实现这个指令功能的逻辑代码，上面的代码实现的是对某个使用该指令元素赋予指令对象值的10倍。

<h2 v-big = "n"></h2>

如果我们在上面的元素使用该指令，这里的element参数就是这里的h2DOM元素，binding参数就是这里的n对象。同样这里定义的是局部的指令，如果想定义全局的指令乐意使用下面的代码：

// 定义全局自定义指令
Vue.directive('fbg',{
  inserted(element) {
       element.style.backgroundColor = "blue"
   }
})

上面的代码中定义了一个全局指令fbg，该指令可以在其他Vue实例中使用。该指令的功能是改变当前元素的背景颜色

<div id="app">
  <div class="box"  v-fbg></div>
</div>

<div id="root">
  <div class="box"  v-fbg></div>
</div>
