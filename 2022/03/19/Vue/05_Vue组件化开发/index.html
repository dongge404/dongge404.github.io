组件分类

非单文件组件

该类型的组件是由多个组件构成，并融合在一个html文件里。先来看一下它的基本使用：

使用一个组件需要进行的操作：

  1. 定义组件（Vue.extend）
  2. 注册组件
    1. 局部注册
    2. 全局注册
  3. 使用组件
    1. 双标签
    2. 单标签
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
  <script src="./js/vue.js"></script>
</head>
<body>
  <div id="app">
    <School></School>
  </div>

  <script>
    // 定义组件  变量school接收一个vc
    const School =  Vue.extend({
      name: "School", // 定义组件的名字
      template:  // 组件模板 用于定义组件中的结构
      `
      <div>
          <h2> {{ schoolName }} </h2>  
      </div>
      `,
      data() {
        return {
          schoolName: "云南大学"
        }
      },
    })
    var vm=new Vue({
      el:'#app',
        // 在vm实例对象中注册School组件
       components: {
        School
      }
    })
  </script>
</body>
</html>

下面代码中定义了三个组件：School、Student、StudentId，其中后两个组件为父子组件：

Root为根组件，即为vm实例对象（本质上可以看成一个组件）

组件中的几个注意事项

  1. 组件命名问题
    1. 写法一： 一个单词全小写（vue开发者工具最终会把这个单词的首字母大写）也可以首字母大写
    2. 写法二： 多个单词组成，可以每个单词首字母大写(-适用于脚手架)或者使用-连接
  2. 组件写法：
    1. 直接使用组件名 <\Myschool></Myschool> 形成闭合标签
    2. 使用一个闭合的标签 <\Myschool /> 不适用脚手架时，会导致该组件的后继组件不能被渲染
  3. 注册组件的简写： const s = Vue.extend(options) = const s = options

组件的嵌套

之前的代码中已经展示过组件的嵌套的写法，在Vue的组件开发中，其实Root根组件（vm）它只管理一个组件，这个组件为App，剩下的所有组件都被App组件所管理：

基于上面的原理图：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
  <script src="./js/vue.js"></script>
</head>
<body>
  <div id="app">
     <App></App>
  </div>

  <script>
    const Student = Vue.extend({
      name: "Student",
      template:
      `
      <div>
        <h2> {{ name }} </h2>
      </div>
      `,
      data() {
        return {
          name: "李明"
        }
      },
    })
    const School =  Vue.extend({
      name: "School", // 定义组件的名字
      template:
      `
      <div>
          <h2> {{ schoolName }} </h2>

      </div>
      `,
      data() {
        return {
          schoolName: "云南大学"
        }
      },
      components: {
        Student
      }
    })
    const Hello = Vue.extend({
      name: "Hello",
      template:
      `
       <div>
          <h2>你好啊！！</h2>

    const App = Vue.extend({
      name: "App",

         School,

      }

      el:'#app',

      }

  </script>
</body>

VueComponent

VueComponent是一个组件实例对象，我们在之前学过的vm，实际上是一个Vue实例对象，他们两个虽然内容和功能上基本相似，但是他们是两个完全独立的东西。

  1. Vue组件本质上是一个名为VueComponent的构造函数, 并不是程序员自己封装的 而是Vue.extend自动生成的
  2. 我们在使用组件<\school></\school>或者<\school />时,Vue在解析这些组件时，会帮助我们生成组件相应的实例对象,即自动调用 new VueComponent(options) 这里的options就是我们在调用extend时传入的{}配置对象中的数据
  3. 特别注意： 每次调用extend都会返回一个全新的vueCComponent组件
console.log("@",school === hello);

如果我们定义两个组件school和hello，那么上面的代码将会输出false，因为在每次调用extend函数时，都会返回一个全新的，完全独立的vc，所以他们本质上是不相等的。

还有，关于this指向问题，需要注意：

  1. new Vue(options)中, data函数 methods函数 watch函数、computed函数 他们的this均指向【Vue实例对象】
  2. 在组件配置中，data函数 methods函数 watch函数、computed函数 他们的this均指向【VueComponent实例对象】
  3. VueComponent以后简称为vc; Vue的实例对象简称vm
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
  <script src="../../js/vue.js"></script>
</head>
<body>
  <div id="app">
   <school></school>
  </div>

  <script>
      const school = Vue.extend({
      name: "school",
      template:
      `
      <div>
       <h2>学校: {{ school }} </h2>  
       <h2>地址: {{ address }} </h2>
       <button @click = "showX">点我输出x</button>
      </div>
      `,
      data() {
        return {
          school: "云南大学",
          address: "呈贡"
        }
      },
      methods: {
        showX() {
          console.log(this.X);  //this为组件的实例对象 通过这个关系来拿到Vue原型对象上的X属性
        }
      }
    })
    var vm=new Vue({
      el:'#app',
      components: {
        school
      }
    })
    Vue.prototype.X = 100   // 在Vue的原型对象上设置X属性

    console.log(school.prototype.__proto__ === Vue.prototype);
    // school（vc）的原型对象上的隐式原型对象和Vue（vm）的原型对象相等
  </script>
</body>
</html>

单文件组件

即整个文件都是一个组件，不存在其他组件，也不是一个html文件的格式，而是一个以.Vue后缀名结尾的文件。单文件组件的执行流程：

1、定义单文件组件Student

<ClientOnly>
<template>
  <div class="demo">
    <h2>学生姓名: {{ name }}</h2>
    <h2>年龄: {{ age }}</h2>
    <button @click="showName">点击显示学校名</button>
  </div>
</template>

<script>
// 组件脚本
// 默认暴露该组件的脚本
export default {
  Student,
}
// 组件的脚本
const student = Vue.extend({
  name: 'Student',
  template: `
      <div>
       <h2>学生: {{ name }} </h2>  
       <h2>年龄: {{ age }} </h2>  
      </div>
      `,
  data() {
    return {
      name: '李华',
      age: 22,
    }
  },
})
</script>

<style>
.demo {
  background-color: yellow;
}
</style>
</ClientOnly>

template：负责组件的结构

script：负责组件的脚本

style：负责组件的样式

2、定义School组件

<template>
  <div class="demo">
    <h2>学校名称: {{ school }}</h2>
    <h2>学校地址: {{ address }}</h2>
    <button @click = "showName">点击显示学校名</button>
  </div>
</template>

<script>
 // 组件脚本
 // 默认暴露该组件的脚本
export default {
  school
}
  // 组件的脚本
  const school = Vue.extend({
      name: "School",
      template:
      `
      <div>
       <h2>学校: {{ school }} </h2>  
       <h2>地址: {{ address }} </h2>
       <button @click = "showName">点我显示学校名</button>
      </div>
      `,
      data() {
        return {
          school: "云南大学",
          address: "呈贡"
        }
      },
      methods: {
        showName() {
          console.log(this.school);
          console.log(this);
        }
      }
    })
</script>

<style>
 .demo {
   background-color: orange;
 }
</style>

3、定义App组件

<template>
  <div>
    <School></School>
    <Student></Student>
  </div>
</template>

<script>
// 引入其他组件
import School from './School.vue'
import Student from './Student.vue'
// 摩恩导出这个App组件的配置对象
export default {
   name: "App",
   // 注册其他组件
   components: {
     School,
     Student
   }
}
</script>

4、定义入口文件main.js

import App from './App.vue'

new Vue({
  el: "#root",
  template: `<App></App>`,
  components: {
    App
  }
})
